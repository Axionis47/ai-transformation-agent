# ADR-003: Vector Store Selection

## Status
Approved

## Context
The RAG agent needs a vector store for company similarity matching.

Dataset size: 20 seed solutions initially, scaling to ~1000 max.

Options:
- ChromaDB (local, embedded)
- ChromaDB on Cloud Storage (load at startup)
- Vertex AI Vector Search (managed)
- Pinecone (external SaaS)

## Decision
**ChromaDB (local)** for Sprint 1 with data persisted to Cloud Storage.

Container startup loads vector store from GCS (~2-3s).

## Rationale
1. 20-1000 documents is tiny — no need for managed service
2. ChromaDB runs in-process with zero network hops
3. Cloud Storage is ~$0/month for this data size
4. Keeps everything in single GCP billing account
5. No external API keys to manage

## Configuration
```bash
VECTOR_STORE=chroma
CHROMA_PERSIST_DIR=./rag/store
```

For production, switch to:
```bash
VECTOR_STORE=chroma
CHROMA_GCS_BUCKET=ai-transform-agent-rag
```

## Embedding Model
Sprint 1: Use ChromaDB's built-in embeddings (all-MiniLM-L6-v2)
Future: Swap to Vertex AI embeddings (textembedding-gecko) if quality insufficient

## Consequences

**Positive:**
- Zero infrastructure cost
- Fast queries (in-memory)
- Simple abstraction (rag/vector_store.py)
- Works offline for development

**Negative:**
- Startup time adds ~2-3s (loading from GCS)
- Container restarts require reload
- Not horizontally scalable (single instance)

## Migration Path
If scale exceeds ~10K documents:
1. Implement rag/vertex_vector_store.py
2. Change VECTOR_STORE=vertex
3. No other code changes (abstraction handles it)
