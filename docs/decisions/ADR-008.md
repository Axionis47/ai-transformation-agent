# ADR-008: Tool Registry as Orchestrator-Level Abstraction

Status: Accepted
Date: 2026-03-16
Decider: Backend (Sprint 5)

---

## Context

Sprint 5 introduced a requirement that Sprint 6 should be able to add new pipeline
tools without modifying `orchestrator/pipeline.py`. Previously, each pipeline stage
imported its agent class directly. Adding a new tool required editing the pipeline
file, creating a risk of regressions and violating the open/closed principle.

---

## Decision

Introduce a `Tool` ABC and a `ToolRegistry` singleton in `orchestrator/tool_registry.py`.

Key decisions:

1. **Tool ABC** — each tool exposes a single `run(input_data: dict) -> dict | AgentError`
   method. Tools wrap agents; agents remain the implementation layer.

2. **ToolRegistry singleton** — tools register by `tool_id` at module import time.
   Pipeline stages call `registry.get("tool_id")` instead of importing agent classes.

3. **tools/ directory** — tool wrapper classes live in `tools/`, separate from
   `agents/`. Agents are not modified. The wrapper handles env var coordination
   (e.g. DRY_RUN flag forwarding).

4. **Scraper migrated first** — `WebsiteScraperTool` wraps `ScraperAgent` and is
   registered by default. `pipeline.py` Stage 1 now uses the registry.

---

## Consequences

Positive:
- Sprint 6 can add a second tool by creating a file in `tools/` and calling
  `registry.register(NewTool())` — zero changes to `pipeline.py`
- Tools are independently testable without importing pipeline logic
- The Tool ABC enforces a consistent interface contract

Negative:
- One extra layer of indirection for the scraper stage
- Module-level auto-registration creates an import-order dependency
  (tools must be importable when `tool_registry` is first imported)

---

## Files changed

- `orchestrator/tool_registry.py` — Tool ABC, ToolRegistry, singleton + auto-registration
- `tools/__init__.py` — tools package
- `tools/website_scraper.py` — WebsiteScraperTool wrapping ScraperAgent
- `orchestrator/pipeline.py` — Stage 1 uses registry instead of direct ScraperAgent import
