# ADR-002: Authentication Strategy

## Status
Approved

## Context
The system needs an authentication strategy for the demo and future production use.

Options considered:
- Firebase Auth (email/password + OAuth)
- Cloud IAP (GCP-native)
- No auth (public demo URL)

## Decision
**No authentication for MVP.** The demo URL will be public.

If auth becomes required post-demo, add Cloud IAP as a day-2 feature (single toggle on Cloud Run).

## Rationale
1. This is an internal diagnostic tool for the Tenex demo
2. No user accounts, sessions, or OAuth needed
3. Adding auth creates friction for evaluators testing the live URL
4. Cloud IAP can be enabled in minutes if needed later

## Consequences

**Positive:**
- Zero auth code to write
- No OAuth provider setup
- Evaluators can test immediately
- Faster time to demo

**Negative:**
- Anyone with URL can run pipeline
- Rate limiting becomes important (consider Cloud Armor)
- No usage tracking per user

## Future Migration Path
If multi-tenant auth is required:
1. Enable Cloud IAP on Cloud Run service
2. Or add Firebase Auth with Google SSO
3. Both are additive changes, no code rewrite needed
