# ADR-005: Secrets Management

## Status
Approved

## Context
The system needs API keys and service account credentials.
These must never be committed to git.

Options:
- Environment variables in Cloud Run config
- Secret Manager with runtime access
- Hashicorp Vault

## Decision
**Secret Manager** for production secrets.
Environment variables from `.env` for local development.

## Secret Inventory

| Secret | Where Used | Storage |
|--------|------------|---------|
| Anthropic API Key | Evals judge | Secret Manager |
| GCP Service Account | Vertex AI calls | Workload Identity (no key) |
| Linear API Key | Ticket automation | Secret Manager |
| Langfuse Keys | Observability | Secret Manager (optional) |

## Local Development
```bash
# .env file (gitignored)
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_APPLICATION_CREDENTIALS=./secrets/service-account.json
```

## Production (Cloud Run)
```bash
# Secrets mounted from Secret Manager
gcloud run services update ai-transform-agent \
  --set-secrets=ANTHROPIC_API_KEY=anthropic-key:latest
```

## Service Account Permissions
```
roles/aiplatform.user              # Vertex AI inference
roles/storage.objectViewer         # Load RAG from GCS
roles/secretmanager.secretAccessor # Read secrets
```

## Consequences

**Positive:**
- Secrets never in git
- Automatic rotation support
- Audit logging of secret access
- Free for small volumes

**Negative:**
- Adds IAM complexity
- Must grant secretAccessor role
- Local dev needs .env file

## CI/CD
GitHub Actions uses Workload Identity Federation:
- No GCP service account key in GitHub secrets
- OIDC authentication to GCP
- Automatic credential rotation
