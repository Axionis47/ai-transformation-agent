# ADR-004: Model Routing Configuration

## Status
Approved

## Context
The pipeline calls multiple LLM endpoints for different tasks.
Need to balance cost, latency, and quality.

## Decision

| Pipeline Stage | Model | Provider | Rationale |
|----------------|-------|----------|-----------|
| Consultant (maturity scoring) | gemini-2.5-pro | Vertex AI | Strong reasoning, few calls |
| Report Writer (5 sections) | gemini-2.5-flash | Vertex AI | Fast, cheap, synthesis task |
| RAG Query (relevance notes) | gemini-2.5-flash | Vertex AI | Simple generation |
| Eval Judge | claude-sonnet-4-20250514 | Anthropic | Best judge model |

## Cost Estimate Per Run
```
Scraper:        $0.00 (no model call)
Consultant:     ~$0.005 (2K input, 1K output)
RAG Query:      ~$0.001
Report Writer:  ~$0.004 (5 sections)
─────────────────────────────
Pipeline Total: ~$0.011

Eval (optional): ~$0.04
```

## Environment Variables
```bash
MODEL_PROVIDER=vertex
VERTEX_MODEL=gemini-2.5-pro
VERTEX_FAST_MODEL=gemini-2.5-flash
GCP_PROJECT_ID=your-project-id
GCP_LOCATION=us-central1
ANTHROPIC_API_KEY=sk-ant-...  # Evals only
```

## Abstraction Layer
All model calls go through `ops/model_client.py`:
- VertexProvider (primary)
- OllamaProvider (local fallback)
- AnthropicProvider (evals only)

## Consequences

**Positive:**
- Single GCP billing for pipeline
- ~$0.01 per run is well under $0.50 budget
- No cold start penalty (Vertex AI is always warm)
- Flash variant cuts report generation time

**Negative:**
- Two providers to manage (Vertex + Anthropic)
- Anthropic key needed for evals
- Provider outage affects full pipeline

## Fallback Strategy
If Vertex AI is unavailable:
1. Set MODEL_PROVIDER=ollama
2. Run qwen2.5:72b on local GPU
3. Or use Anthropic as fallback (higher cost)
