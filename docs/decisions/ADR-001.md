# ADR-001: Infrastructure Stack for MVP

## Status
Approved

## Context
The AI Transformation Discovery Agent needs infrastructure before Sprint 1.

Key constraints:
- 90-second end-to-end runtime
- $0.50 max cost per run
- Must work for Tenex demo
- No over-engineering

## Decision

| Component | Technology | Rationale |
|-----------|------------|-----------|
| Backend Hosting | Cloud Run | Serverless, scales to zero, 300s timeout, no cluster management |
| Frontend Hosting | Vercel | Free tier, instant deploys, optimal for Next.js |
| Container Registry | Artifact Registry | Same GCP project, integrated with Cloud Build |
| CI/CD | Cloud Build + GitHub Actions | Actions for lint/test, Cloud Build for deploy |

## Consequences

**Positive:**
- Zero infrastructure to manage
- Pay only when invoked
- HTTPS out of the box
- Automatic scaling

**Negative:**
- 300s timeout limit (sufficient for 90s pipeline)
- Cold start latency (~2-3s first request)
- No persistent filesystem (must use Cloud Storage for RAG)

## Alternatives Considered

| Alternative | Why Rejected |
|-------------|--------------|
| GKE Autopilot | Overkill for demo scale, adds operational burden |
| GCE with Docker | Persistent VMs cost money when idle |
| AWS Lambda | Would split billing across providers |
| Fly.io | Good option, but sticking with single GCP provider |
