# Component Map

What every file in this repo does. Generated by reading every source file on 2026-03-17.

## Entry Point

### app.py
Purpose: FastAPI application - exposes POST /v1/analyze, GET /v1/trace/{run_id}, and GET /health.
Exports: `app` (FastAPI instance), `AnalyzeRequest`, `AnalyzeSuccess`, `AnalyzeError`, `ErrorDetail`
Input: HTTP requests - POST body `{ url, dry_run }` for /v1/analyze; `run_id` path param for /v1/trace
Output: JSON response with run_id, status, elapsed_seconds, cost_usd, report, maturity, use_cases, victory_matches
Depends on: `orchestrator.pipeline`, `orchestrator.state`, `infra.health_check`

## Agents (agents/)

### agents/base.py
Purpose: Abstract base class and error type that all pipeline agents must implement.
Exports: `AgentError` (dataclass), `BaseAgent` (ABC)
Input: Any dict passed to `run(input_data)`
Output: `AgentError` on any unhandled exception, or whatever `_run` returns
Depends on: nothing in the repo

### agents/scraper.py
Purpose: Scrapes a company URL (about, careers, product pages) and returns structured text.
Exports: `ScraperAgent`
Input: `{ url: str }` - in dry-run mode reads from `tests/fixtures/sample_company.json`
Output: `{ url, name, about_text, job_postings, product_text, tech_stack_mentions, pages_fetched }`
Depends on: `agents.base`, `httpx` (live), `beautifulsoup4` (live)

### agents/consultant.py
Purpose: Calls Vertex AI (Gemini Pro) to score AI maturity and identify use cases from scraped data + RAG context.
Exports: `ConsultantAgent`, `format_wins`
Input: `{ company_data: dict, rag_context: list[dict] }` - in dry-run reads `tests/fixtures/sample_analysis.json`
Output: JSON dict with maturity assessment and use cases (matches AnalysisResult schema in prompt)
Depends on: `agents.base`, `ops.model_client`, `prompts/consultant.md`

### agents/rag_query.py
Purpose: Queries the vector store for Tenex victory records similar to the company being analyzed.
Exports: `RAGQueryAgent`, `_INDUSTRY_KEYWORDS`
Input: `{ company_data: dict }` - builds a structured query from about_text + job_postings
Output: `list[dict]` - up to 3 matching victory records; in dry-run returns first 3 from victories.json
Depends on: `agents.base`, `rag.vector_store`

### agents/signal_extractor.py
Purpose: Calls Gemini Flash to extract typed signals (tech stack, ML signals, intent, scale) from raw company text.
Exports: `SignalExtractorAgent`
Input: `{ company_data: dict }` - combines about, job, and product text into a structured prompt
Output: JSON matching `SignalSet` schema - in dry-run reads `tests/fixtures/sample_signals.json`
Depends on: `agents.base`, `ops.model_client`, `prompts/signal_extractor.md`

### agents/maturity_scorer.py
Purpose: Calls Gemini Flash to score four AI maturity dimensions from typed signals.
Exports: `MaturityScorerAgent`
Input: `{ signals: dict }` - the SignalSet output from signal_extractor
Output: JSON matching `MaturityResult` schema - in dry-run reads `tests/fixtures/sample_maturity.json`
Depends on: `agents.base`, `ops.model_client`, `prompts/maturity_scorer.md`

### agents/report_writer.py
Purpose: Calls Gemini Flash to generate the 5-section AI transformation report from analysis data.
Exports: `ReportWriterAgent`, `REPORT_SECTIONS`
Input: `{ analysis: dict }` containing maturity, signals, use_cases, victory_matches
Output: JSON with keys exec_summary, current_state, use_cases, roadmap, roi_analysis
Depends on: `agents.base`, `ops.model_client`, `prompts/report_writer.md`

### agents/use_case_generator.py
Purpose: Calls Gemini Pro to generate tiered AI use case recommendations from signals, maturity, and victories.
Exports: `UseCaseGeneratorAgent`
Input: `{ signals, maturity, victory_matches }` - in dry-run reads `tests/fixtures/sample_use_cases.json`
Output: JSON array of UseCase objects in three tiers (LOW_HANGING_FRUIT, MEDIUM_SOLUTION, HARD_EXPERIMENT)
Depends on: `agents.base`, `ops.model_client`, `prompts/use_case_generator.md`

## Orchestrator (orchestrator/)

### orchestrator/pipeline.py
Purpose: Runs the 7-stage pipeline: scrape, signal extract, maturity score, RAG query, victory match, use case generate, report write.
Exports: `run_pipeline`, `print_report`, `main`
Input: `url: str, dry_run: bool`
Output: `PipelineState` with all stage outputs populated
Depends on: all agent modules, `orchestrator.state`, `orchestrator.schemas`, `orchestrator.gates`, `orchestrator.stage_io`, `orchestrator.tool_registry`, `orchestrator.validators`, `orchestrator.victory_matcher`, `ops.logger`, `rag.ingest`

### orchestrator/state.py
Purpose: Typed working memory for a single pipeline run - passed between all pipeline stages.
Exports: `PipelineState`, `PipelineStatus`
Input: `url: str, dry_run: bool` at construction; each stage populates its output fields
Output: Fully populated state after pipeline completes
Depends on: `pydantic`

### orchestrator/schemas.py
Purpose: Pydantic models for all typed artifacts that flow between pipeline stages.
Exports: `Signal`, `SignalSet`, `DimensionScore`, `MaturityResult`, `VictoryMatch`, `DataFlow`, `UseCase`
Input: Dicts from agent outputs
Output: Validated typed objects
Depends on: `pydantic`

### orchestrator/gates.py
Purpose: Deterministic content quality check that runs after scraping - blocks thin or error-page responses.
Exports: `scraper_quality_gate`
Input: `company_data: dict` from ScraperAgent
Output: `tuple[bool, str]` - (passed, reason)
Depends on: nothing in the repo

### orchestrator/validators.py
Purpose: Validates and normalizes agent outputs against Pydantic schemas before writing to PipelineState.
Exports: `validate_signals`, `validate_maturity`, `validate_use_cases`
Input: Raw dicts from agent outputs
Output: Typed Pydantic objects or `AgentError`
Depends on: `agents.base`, `orchestrator.schemas`

### orchestrator/victory_matcher.py
Purpose: Deterministic scoring that classifies RAG results into DIRECT, CALIBRATION, or ADJACENT match tiers without a model call.
Exports: `match_victories`, `_proximity_score`, `_industries_related`
Input: `signals_industry, signals_scale, maturity_label, rag_results, company_composite_score`
Output: `list[VictoryMatch]` sorted by tier and similarity score
Depends on: `orchestrator.schemas`

### orchestrator/stage_io.py
Purpose: Builds structured input/output summary dicts for each pipeline stage - used by the logger.
Exports: `scraper_input`, `scraper_output`, `signal_input`, `signal_output`, `maturity_input`, `maturity_output`, `rag_input`, `rag_output`, `victory_input`, `victory_output`, `use_case_input`, `use_case_output`, `report_writer_input`, `report_writer_output`
Input: Stage data from PipelineState fields
Output: Small dicts suitable for JSONL log entries
Depends on: nothing in the repo

### orchestrator/tool_registry.py
Purpose: Abstract Tool contract and ToolRegistry singleton that decouples pipeline stages from concrete agent classes.
Exports: `Tool` (ABC), `ToolRegistry`, `registry` (singleton)
Input: Tool instances registered via `registry.register(tool)`
Output: Tool instances retrieved via `registry.get(tool_id)`
Depends on: `agents.base`, `tools.website_scraper` (registered at module load)

## Ops (ops/)

### ops/model_client.py
Purpose: Single abstraction layer for all LLM calls - never call model providers directly.
Exports: `ModelClient` (ABC), `VertexProvider`, `MockProvider`, `get_model_client`
Input: `prompt: str, system: str, model: str` passed to `complete()`
Output: `str` response text or `AgentError`
Depends on: `agents.base`, `google-cloud-aiplatform` (Vertex), env vars

### ops/logger.py
Purpose: Appends structured JSONL log entries to per-run log files in logs/runs/{run_id}.jsonl.
Exports: `PipelineLogger`, `get_logger`
Input: `agent_tag, event, input_summary, output_summary` and arbitrary kwargs
Output: Appended JSONL lines to disk - never returned to callers
Depends on: nothing in the repo

## RAG (rag/)

### rag/vector_store.py
Purpose: Single abstraction layer for all vector store operations - ChromaDB backend with MockStore for dry-run.
Exports: `VectorStore` (ABC), `ChromaStore`, `MockStore`, `get_vector_store`
Input: `add(docs: list[dict])` or `query(text: str, k: int)`
Output: `None` on add, `list[dict]` on query, or `AgentError` on failure
Depends on: `agents.base`, `chromadb`, env vars

### rag/ingest.py
Purpose: Loads Tenex victory records from fixtures into ChromaDB at pipeline startup - idempotent.
Exports: `ensure_seeds_loaded`
Input: Reads from `tests/fixtures/rag_seeds/victories.json` (or seeds.json as fallback)
Output: ChromaDB upsert (side effect only) - skips if already seeded
Depends on: `rag.vector_store`, `chromadb`

Exports: `DeployResult` (dataclass), `DeployTarget` (ABC), `GCPCloudRunTarget`, `get_deploy_target`

Input: GET /health request
Output: JSON `{ status, version, model_provider, pipeline }` with HTTP 200
Depends on: `fastapi`, env vars

## Evals (evals/)

### evals/judge_client.py
Purpose: Scores pipeline outputs against YAML rubrics using Gemini Pro on Vertex AI as judge.
Exports: `JudgeClient`
Input: `rubric_path: str, output: dict` - rubric must have a `judge_prompt_template` key
Output: `float` score 1.0-5.0 (returns 0.0 on error or unavailability)
Depends on: `google-cloud-aiplatform`, `yaml`, env vars

### evals/ci_eval.py
Purpose: Baseline eval runner - runs dry-run pipeline per test company, scores outputs, writes to baselines.json.
Exports: `run_baseline`
Input: Reads `evals/test_companies.json`, runs pipeline, loads rubrics from `evals/rubrics/`
Output: Updates `evals/baselines.json` with sprint-keyed results and averages
Depends on: `evals.judge_client`, `orchestrator.pipeline`

### evals/rubrics/tier_classification.yaml
Purpose: YAML rubric for judging whether use case tiers (LOW/MEDIUM/HARD) match signal evidence.
Exports: `judge_prompt_template` field consumed by JudgeClient
Depends on: nothing in the repo

### evals/rubrics/evidence_grounding.yaml
Purpose: YAML rubric for judging whether use case descriptions cite specific signals from the analysis.
Exports: `judge_prompt_template` field consumed by JudgeClient
Depends on: nothing in the repo

### evals/rubrics/roi_basis.yaml
Purpose: YAML rubric for judging whether ROI estimates have a stated quantitative basis.
Exports: `judge_prompt_template` field consumed by JudgeClient
Depends on: nothing in the repo
